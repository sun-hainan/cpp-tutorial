## 1. 设计模式：单例、工厂、代理

### 生活类比

> - **单例模式**像**唯一的总统**——一个国家只有一个总统，不管谁想当都要找同一个人
> - **工厂模式**像**工厂流水线**——你下单（参数），工厂生产出对应的产品
> - **代理模式**像**房产中介**——你不直接找房东，通过中介代理处理

### 单例模式

```cpp
class Singleton {
public:
    static Singleton& instance() {
        static Singleton inst;  // C++11 保证线程安全
        return inst;
    }

    void set_value(int v) { value_ = v; }
    int get_value() const { return value_; }

    Singleton(const Singleton&) = delete;
    Singleton& operator=(const Singleton&) = delete;

private:
    Singleton() : value_(0) { }
    int value_;
};

Singleton& s1 = Singleton::instance();
Singleton& s2 = Singleton::instance();
s1.set_value(42);
std::cout << s2.get_value() << std::endl;  // 42
```

### 简单工厂

```cpp
class Product {
public:
    virtual ~Product() = default;
    virtual std::string operation() const = 0;
};

class ConcreteProductA : public Product {
public:
    std::string operation() const override { return "ProductA"; }
};

class ConcreteProductB : public Product {
public:
    std::string operation() const override { return "ProductB"; }
};

class SimpleFactory {
public:
    std::unique_ptr<Product> create(const std::string& type) {
        if (type == "A") return std::make_unique<ConcreteProductA>();
        if (type == "B") return std::make_unique<ConcreteProductB>();
        return nullptr;
    }
};
```

### 代理模式

```cpp
class RealService {
public:
    std::string do_work() { return "RealService doing work"; }
};

class ProxyService {
public:
    ProxyService() : real_(std::make_unique<RealService>()) { }

    std::string do_work() {
        std::cout << "代理：权限检查..." << std::endl;  // 前置处理
        std::string result = real_->do_work();
        std::cout << "代理：日志记录..." << std::endl;  // 后置处理
        return result;
    }

private:
    std::unique_ptr<RealService> real_;
};
```

## 2. 策略模式与观察者模式

### 策略模式

> **策略模式**像**游戏手柄的按键映射**——同一个按键可以映射到不同动作（策略），随时可以切换。

```cpp
class SortStrategy {
public:
    virtual ~SortStrategy() = default;
    virtual std::vector<int> sort(std::vector<int> data) const = 0;
};

class QuickSortStrategy : public SortStrategy {
public:
    std::vector<int> sort(std::vector<int> data) const override {
        std::sort(data.begin(), data.end());
        return data;
    }
};

class Context {
public:
    void set_strategy(std::unique_ptr<SortStrategy> s) { strategy_ = std::move(s); }
    std::vector<int> execute_sort(const std::vector<int>& data) {
        return strategy_->sort(std::vector<int>(data));
    }
private:
    std::unique_ptr<SortStrategy> strategy_;
};
```

### 观察者模式

> **观察者模式**像**订阅报纸**——你订阅了报纸，每当有新报纸出版（事件），就会自动送到你家。

```cpp
class Observer {
public:
    virtual ~Observer() = default;
    virtual void update(const std::string& msg) = 0;
};

class Subject {
public:
    void attach(std::shared_ptr<Observer> obs) { observers_.push_back(obs); }
    void notify(const std::string& msg) {
        for (auto& obs : observers_) obs->update(msg);
    }
private:
    std::vector<std::shared_ptr<Observer>> observers_;
};
```

## 3. SOLID 六大原则

| 原则 | 全称 | 说明 |
|------|------|------|
| S | Single Responsibility | 单一职责：类只做一件事 |
| O | Open/Closed | 开闭原则：对扩展开放，对修改封闭 |
| L | Liskov Substitution | 里氏替换：子类能替换基类 |
| I | Interface Segregation | 接口隔离：多个专用接口优于通用接口 |
| D | Dependency Inversion | 依赖反转：依赖抽象而非具体 |
| DRY | Don't Repeat Yourself | 不重复原则：相同逻辑不重复 |

### KISS & YAGNI

| 原则 | 说明 |
|------|------|
| KISS | Keep It Simple, Stupid - 保持简单 |
| YAGNI | You Aren't Gonna Need It - 不要为将来可能用不到的功能提前实现 |

## 4. CMake 基础

### 基本结构

```cmake
cmake_minimum_required(VERSION 3.14)
project(MyProject VERSION 1.0 LANGUAGES CXX)

# C++ 标准
set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

# 头文件目录
include_directories(include)

# 源文件目录
aux_source_directory(src SRC_LIST)

# 生成可执行文件
add_executable(myapp ${SRC_LIST})

# 生成库
add_library(mylib STATIC src/lib.cpp)
target_link_libraries(myapp PRIVATE mylib)

# 测试
enable_testing()
add_subdirectory(tests)

# 安装
install(TARGETS myapp DESTINATION bin)
```

### 常用命令

```bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build
cmake --install build
```

## 6. 命名空间

### 基本使用

```cpp
namespace project {
    namespace module_a {
        class Config {
        public:
            void init() { std::cout << "module_a::Config::init" << std::endl; }
        };
    }
}

project::module_a::Config ca;
ca.init();

// using 声明
using project::module_a::Config;
Config c;  // 不需要完整路径

// 命名空间别名
namespace alias = project::module_a;
alias::Config* cap = &ca;
```

### 最佳实践

```
✅ 避免 using namespace std;（尤其在头文件）
✅ 头文件用完整命名空间
✅ 实现文件用 using 简化
```
