## 1. 类与对象：面向对象的核心

### 生活类比

> **类**就像**建筑图纸**——定义了房子的结构、房间布局、门窗位置。
> **对象**就是根据图纸**建造出来的房子**——每一栋都是独立的实例，但共享同样的设计。

### 概念讲解

```cpp
class Point {
public:
    // 默认构造
    Point() : x_(0), y_(0) { }

    // 参数构造
    Point(int x, int y) : x_(x), y_(y) { }

    // C++11 委托构造
    Point(int val) : Point(val, val) { }

    // C++11 列表初始化
    Point(std::initializer_list<int> il)
        : x_(*il.begin()), y_(*(il.begin()+1)) { }

    void print() const { std::cout << "Point(" << x_ << "," << y_ << ")" << std::endl; }

private:
    int x_, y_;
};

// 创建对象
Point p1;           // 默认构造
Point p2(3, 4);     // 参数构造
Point p3(5);        // 委托构造
Point p4 = {6, 7};  // 列表初始化
```

## 2. 构造、析构、拷贝、移动

### 生活类比

> - **构造函数**像**入住登记**——你搬进来时，系统记录你的信息
> - **析构函数**像**退房手续**——你离开时，系统清理你的记录
> - **拷贝构造**像**复印文件**——创建一份一模一样的新文件
> - **移动构造**像**搬家**——把东西直接搬走，不复印

### 概念讲解

| 操作 | 触发时机 | 默认行为 |
|------|---------|---------|
| 默认构造 | `T obj;` | 成员默认值初始化 |
| 拷贝构造 | `T obj2 = obj1;` | 逐成员拷贝 |
| 移动构造 | `T obj2 = std::move(obj1);` | 窃取资源 |
| 拷贝赋值 | `obj2 = obj1;` | 逐成员拷贝赋值 |
| 移动赋值 | `obj2 = std::move(obj1);` | 窃取资源赋值 |
| 析构函数 | 对象销毁时 | 清理资源 |

### 代码示例

```cpp
class Resource {
public:
    Resource() : data_(new int[100]) {
        std::cout << "默认构造，分配资源" << std::endl;
    }

    ~Resource() {
        delete[] data_;
        std::cout << "析构函数，释放资源" << std::endl;
    }

    // 拷贝构造（深拷贝）
    Resource(const Resource& other) : data_(new int[100]) {
        for (int i = 0; i < 100; ++i) data_[i] = other.data_[i];
        std::cout << "拷贝构造（深拷贝）" << std::endl;
    }

    // 移动构造（C++11）
    Resource(Resource&& other) noexcept : data_(other.data_) {
        other.data_ = nullptr;  // 转移所有权
        std::cout << "移动构造（所有权转移）" << std::endl;
    }
};
```

### 规则补充

```cpp
// C++11 = default / = delete
Resource() = default;                    // 使用编译器生成
Resource(const Resource&) = delete;       // 禁用拷贝
```

## 3. 继承的访问控制

### 生活类比

> **公有继承**像儿子公开宣布"我是爸爸的儿子"——is-a 关系。
> **私有继承**像儿子偷偷学爸爸的手艺——has-a 关系（组合）。
> **保护继承**介于两者之间。

### 概念讲解

| 继承方式 | 基类 public | 基类 protected | 基类 private |
|---------|-------------|----------------|--------------|
| public | → public | → protected | → private |
| protected | → protected | → protected | → private |
| private | → private | → private | → private |

### 代码示例

```cpp
class Animal {
protected:
    std::string name_;  // 派生类可访问
};

// 公有继承：is-a 关系
class Dog : public Animal {
public:
    Dog(const std::string& name) : Animal(name) { }
    void speak() const {
        std::cout << name_ << " 汪汪！" << std::endl;
    }
};

// 私有继承：has-a 关系（类似组合）
class SmartDog : private Animal {
public:
    void guard() const { speak_loud(); }  // 需要通过接口暴露
    void speak_loud() const { speak(); }
};

// 多态通过基类指针
Animal* animal_ptr = new Dog("旺财");
animal_ptr->speak();  // 调用 Dog::speak（虚函数）
```

## 4. 虚函数与多态

### 生活类比

> **虚函数**就像一个**遥控器**——同一个遥控器（基类指针）可以控制不同的电器（派生类对象），按下按钮（调用方法）时执行电器自己的功能。

### 概念讲解

**多态** = 通过基类指针/引用调用派生类的实现

```cpp
class Shape {
public:
    virtual ~Shape() = default;

    // 纯虚函数 → 抽象类
    virtual double area() const = 0;
    virtual void describe() const {
        std::cout << "图形" << std::endl;
    }
};

class Circle : public Shape {
public:
    Circle(double r) : radius_(r) { }

    double area() const override {
        return 3.14159 * radius_ * radius_;
    }

    void describe() const override {
        std::cout << "圆，半径=" << radius_ << std::endl;
    }

private:
    double radius_;
};
```

### 虚函数原理

```
对象内存布局：
┌─────────────────────┐
│ vptr (虚表指针)  ────┼──► 虚表 (vtable)
├─────────────────────┤
│ 成员变量              │
└─────────────────────┘

虚表内容：
┌─────────────────────┐
│ 0: Shape::area      │ ← Circle 重写
│ 1: Circle::describe  │ ← Circle 重写
│ 2: Shape::~Shape    │
└─────────────────────┘

调用虚函数：obj.vptr[index](&obj) → 间接跳转
```

## 5. static、const、mutable

### 概念讲解

| 修饰符 | 含义 | 示例 |
|--------|------|------|
| static | 类级别共享 | `static int count_;` |
| const | 不可变 | `void foo() const;` |
| mutable | const 中的可变 | `mutable int cache_;` |

### 代码示例

```cpp
class Counter {
public:
    Counter() { ++count_; }

    static int count_;        // 所有对象共享
    static int get_count() {  // 静态成员函数
        return count_;
    }

    void set_value(int v) const {  // const 成员函数
        // value_ = v;  // ❌ 编译错误
        // 但可以用 mutable
    }

    void set_cached(int v) const {
        mutable int* hack = const_cast<int*>(&value_);  // 不推荐！
        *hack = v;  // 技术上可以绕过 const
    }

private:
    int value_ = 0;
};
int Counter::count_ = 0;  // 静态成员定义
```

## 6. 抽象类与接口

### 生活类比

> **抽象类**就像一个**模具**——你可以有模具的样子，但模具本身不能做出实物。
> **接口**是更纯粹的模具——只有形状，没有任何具体实现。

### 概念讲解

```cpp
// C++ 中"接口"约定：用纯虚函数的抽象类
class IDrawable {
public:
    virtual ~IDrawable() = default;
    virtual void draw() const = 0;  // 纯虚函数
    virtual void resize(double factor) = 0;
};

class IPlayable {
public:
    virtual ~IPlayable() = default;
    virtual void play() const = 0;
    virtual void stop() const = 0;
};

// 多接口实现
class VideoPlayer : public IDrawable, public IPlayable {
public:
    void draw() const override { std::cout << "绘制视频" << std::endl; }
    void resize(double factor) override { }
    void play() const override { std::cout << "播放视频" << std::endl; }
    void stop() const override { std::cout << "停止播放" << std::endl; }
};
```

## 7. 嵌套类与局部类

### 概念讲解

```cpp
class Outer {
public:
    Outer(int val) : value_(val) { }

    // 嵌套类
    class Inner {
    public:
        Inner(Outer* o) : outer_(o) { }
        void access_outer() const {
            // 可以访问外部类的所有成员（包括 private）
            std::cout << "Inner 访问外部类" << std::endl;
        }
    };

    Inner create_inner() { return Inner(this); }

private:
    int value_;
};

// 局部类（定义在函数内部）
auto make_local_class = []() {
    class Local {
    public:
        int value;
        void print() const {
            std::cout << "局部类 value=" << value << std::endl;
        }
    };
    return Local{123};
};
auto local_obj = make_local_class();
local_obj.print();
```

## 本章小结

| 概念 | 核心要点 |
|------|---------|
| 类与对象 | 类是蓝图，对象是实例 |
| 构造/析构 | 初始化/清理资源 |
| 拷贝/移动 | 深拷贝 vs 所有权转移 |
| 继承方式 | public=is-a, private=has-a |
| 虚函数 | 通过 vtable 实现运行时多态 |
| override | C++11 明确标记重写 |
| static | 类级别共享，无 this 指针 |
| const | 承诺不修改对象 |
| 抽象类 | 含纯虚函数的类，不能实例化 |
| 接口 | 用纯虚函数抽象类模拟 |
