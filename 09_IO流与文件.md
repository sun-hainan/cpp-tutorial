## 1. iostream 基本使用

### 生活类比

> **iostream** 就像邮局的**收发窗口**——`cout` 是发件窗口，`cin` 是收件窗口。数据像信件一样在程序和外界的"邮局"之间流动。

### 概念讲解

| 流 | 用途 | 缓冲区 |
|---|------|--------|
| std::cout | 标准输出 | 有缓冲 |
| std::cerr | 标准错误 | 无缓冲 |
| std::clog | 标准日志 | 有缓冲 |

### 代码示例

```cpp
// 基本输出
std::cout << "Hello " << "C++" << "!" << std::endl;
std::cerr << "错误信息（无缓冲）" << std::endl;
std::clog << "日志信息（有缓冲）" << std::endl;

// 格式化输出
std::cout << std::hex << 42 << std::endl;      // 十六进制
std::cout << std::oct << 42 << std::endl;      // 八进制
std::cout << std::dec;                          // 恢复十进制

std::cout << std::setw(10) << std::setfill('0') << 42 << std::endl;
std::cout << std::setprecision(3) << std::fixed << 3.1415926 << std::endl;

// 布尔输出
std::cout << std::boolalpha << true << " / " << false << std::endl;
```

### endl vs "\n"

```
endl → 输出换行符 + 刷新缓冲区（强制显示）
"\n" → 只输出换行符（缓冲积累后统一刷新，更高效）
```

## 2. 文件读写

### 生活类比

> **文件读写**就像**搬家公司的货车**——把东西（数据）装进货车（写入），或从货车里卸货（读取）。

### 概念讲解

| 模式 | 说明 | 文件不存在 |
|------|------|-----------|
| std::ios::in | 读取 | 报错 |
| std::ios::out | 写入 | 创建 |
| std::ios::binary | 二进制模式 | - |
| std::ios::app | 追加模式 | 创建 |

### 代码示例

```cpp
// ============ 文本文件写入 ============
std::ofstream out("test.txt");
out << "Line 1: Hello" << std::endl;
out << "Line 2: " << 42 << std::endl;
out.close();

// ============ 文本文件读取 ============
std::ifstream in("test.txt");
std::string line;
while (std::getline(in, line)) {  // 按行读取
    std::cout << line << std::endl;
}
in.close();

// ============ 二进制文件写入 ============
std::ofstream bout("test.bin", std::ios::binary);
int arr[] = {1, 2, 3, 4, 5};
bout.write(reinterpret_cast<const char*>(arr), sizeof(arr));
bout.close();

// ============ 二进制文件读取 ============
std::ifstream bin("test.bin", std::ios::binary);
int read_arr[5];
bin.read(reinterpret_cast<char*>(read_arr), sizeof(read_arr));
bin.close();

// ============ fstream（可同时读写） ============
std::fstream fs("test.txt", std::ios::in | std::ios::out);
fs << "追加内容";
fs.seekg(0);  // 定位到文件开头
std::getline(fs, line);
```

### 文本 vs 二进制模式

```
文本模式：
  写入 "\n" → 实际 "\r\n"（Windows）
  读取 "\r\n" → "\n"（Windows）
  其他平台无转换

二进制模式：
  无任何转换
  适合：图片、音频、结构体序列化
```

## 3. stringstream：内存中的 IO

### 生活类比

> **stringstream** 就像一张**便签纸**——可以先把东西写在纸上（格式化），再决定怎么处理（发送/读取）。

### 概念讲解

| 类型 | 用途 |
|------|------|
| std::ostringstream | 构造字符串（输出到字符串） |
| std::istringstream | 解析字符串（从字符串输入） |

### 代码示例

```cpp
// 格式化字符串
std::ostringstream oss;
oss << "Name: " << "Alice" << ", Age: " << 25
    << ", Score: " << 95.5 << std::endl;
std::string formatted = oss.str();

// 解析字符串
std::string input = "100 3.14 hello";
std::istringstream iss(input);
int i; double d; std::string s;
iss >> i >> d >> s;  // i=100, d=3.14, s="hello"

// 类型转换
int val = 42;
std::ostringstream t; t << val;
std::string str_val = t.str();

std::string str_num = "3.14";
std::istringstream p(str_num);
double pi; p >> pi;
```

## 4. 缓冲区：IO 的"中转站"

### 生活类比

> **缓冲区**就像**快递中转站**——不是来一件发一件，而是先集中一堆再统一发货，提高效率。

### 概念讲解

| 缓冲区类型 | 应用场景 |
|-----------|---------|
| 全缓冲 | 文件 IO |
| 行缓冲 | 终端（stdout/stdin） |
| 无缓冲 | cerr（立即显示） |

### 代码示例

```cpp
// cout 的缓冲区
std::cout << "输出（无换行，不刷新）";
// std::cout.flush();  // 手动刷新

// endl vs "\n"
std::cout << "endl 换行并刷新: " << std::endl;  // 刷新缓冲
std::cout << "\\n 只换行\n";                      // 不刷新

// 同步点
// std::ios_base::sync_with_stdio(false);  // 关闭同步，性能更高
// std::cin.tie(nullptr);                  // cin 不自动 tie cout
```

## 5. 简单序列化

### 生活类比

> **序列化**就像**打包行李**——把各种东西（结构体）装进箱子里（字节流），邮寄到目的地后再**拆箱**（反序列化）恢复原样。

### 概念讲解

```cpp
struct Person {
    char name[32];
    int age;
    double score;
};

// 序列化（写入）
Person p1 = {};
std::strncpy(p1.name, "Alice", sizeof(p1.name) - 1);
p1.age = 25;
p1.score = 95.5;

std::ofstream out("person.bin", std::ios::binary);
out.write(reinterpret_cast<const char*>(&p1), sizeof(p1));
out.close();

// 反序列化（读取）
std::ifstream in("person.bin", std::ios::binary);
Person p2;
in.read(reinterpret_cast<char*>(&p2), sizeof(p2));
in.close();
```

### 注意

```
⚠️ 二进制序列化的坑：
  1. 依赖内存布局，跨平台可能有问题
  2. 结构体可能有对齐/填充字节
  3. 跨平台推荐：protobuf、msgpack、JSON
```

## 6. IO 错误处理

### 代码示例

```cpp
// 检查流状态
std::istringstream iss("42");
int val;
if (iss >> val) {
    std::cout << "解析成功: " << val << std::endl;
}

// 解析失败时流状态
std::istringstream bad("abc");
if (!(bad >> val)) {
    std::cout << "解析 'abc' 失败" << std::endl;
}

// 文件操作错误检查
std::ifstream file("nonexistent.txt");
if (!file) {
    std::cout << "文件不存在或无法打开" << std::endl;
}

// 完整读取检测
std::istringstream data("1 2 3");
while (data >> num) {
    std::cout << "读取数字: " << num << std::endl;
}
if (!data.eof()) {
    std::cout << "读取中断（非 EOF 原因）" << std::endl;
}

// 异常模式
std::ostringstream out;
out.exceptions(std::ios::failbit | std::ios::badbit);
try {
    out << "test";
} catch (const std::ios_base::failure& e) {
    std::cout << "IO exception: " << e.what() << std::endl;
}
```

## 本章小结

| 概念 | 核心要点 |
|------|---------|
| iostream | cout（标准输出）、cin（标准输入）、cerr（无缓冲错误） |
| endl | 输出换行符并刷新缓冲区 |
| 文件流 | ofstream（写）、ifstream（读）、fstream（读写） |
| 二进制模式 | ios::binary，无换行符转换 |
| stringstream | ostringstream（构造字符串）、istringstream（解析字符串） |
| 缓冲区 | 提高 IO 效率，sync_with_stdio(false) 可加速 |
| 序列化 | 二进制序列化依赖内存布局，跨平台用专业库 |
| 错误处理 | 检查 failbit/eofbit，或用 exceptions() 模式 |
