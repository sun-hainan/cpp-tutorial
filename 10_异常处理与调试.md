## 1. 异常处理：程序的"安全网"

### 生活类比

> **异常处理**就像飞机的**紧急降落伞**——平时不用，但遇到紧急情况（异常）时能安全着陆，而不是坠毁。

### 概念讲解

```cpp
auto divide = [](int a, int b) -> double {
    if (b == 0) {
        throw std::runtime_error("除数不能为零");
    }
    return static_cast<double>(a) / b;
};

try {
    double result = divide(10, 2);
    std::cout << "10/2 = " << result << std::endl;
    result = divide(10, 0);  // 抛出异常
} catch (const std::runtime_error& e) {
    std::cout << "捕获异常: " << e.what() << std::endl;
} catch (const std::exception& e) {
    std::cout << "其他异常: " << e.what() << std::endl;
} catch (...) {
    std::cout << "未知异常" << std::endl;  // 最后防线
}
```

### 栈展开原理

```
异常抛出后：
  函数A → 函数B → 函数C（抛出异常）
      ↓         ↓         ↓
    继续展开   继续展开   调用 ~C()（析构）
                              ↓
                            调用 ~B()
                              ↓
                            调用 ~A()
                              ↓
                         到达 catch
```

## 2. 标准异常类层次

### 概念讲解

```
std::exception
├── std::logic_error          ← 程序员的错误
│   ├── std::invalid_argument
│   ├── std::length_error
│   └── std::out_of_range
├── std::runtime_error        ← 运行时错误
│   ├── std::overflow_error
│   ├── std::underflow_error
│   └── std::range_error
├── std::bad_alloc           ← new 失败
├── std::bad_cast            ← dynamic_cast 失败
└── std::bad_exception
```

### 自定义异常

```cpp
class MyException : public std::runtime_error {
public:
    MyException(const std::string& msg, int code)
        : std::runtime_error(msg), code_(code) { }
    int code() const { return code_; }
private:
    int code_;
};

try {
    throw MyException("业务错误", 1001);
} catch (const MyException& e) {
    std::cout << "自定义异常: " << e.what() << ", code=" << e.code() << std::endl;
}
```

## 3. RAII 与异常安全

### 生活类比

> **RAII** 就像**酒店的自动退房系统**——不管客人是正常退房还是遇到紧急情况离店（异常），房间都会自动清理干净。

### 概念讲解

```cpp
class ResourceGuard {
public:
    ResourceGuard() { std::cout << "资源获取" << std::endl; }
    ~ResourceGuard() {
        std::cout << "资源释放（即使异常也会调用）" << std::endl;
    }
    void may_throw() { throw std::runtime_error("工作中出错"); }
};

void demo() {
    ResourceGuard rg;
    try {
        rg.may_throw();  // 抛异常
        std::cout << "这行不会执行" << std::endl;
    } catch (...) {
        std::cout << "异常被捕获" << std::endl;
    }
    // 析构函数仍然被调用！资源安全释放
}
```

### noexcept 的影响

```cpp
// noexcept 声明函数不抛异常
void safe_func() noexcept { }

// 如果 noexcept 函数抛异常，std::terminate() 被调用
// noexcept 让编译器生成更小更快的代码（跳过栈展开）
```

## 4. GDB/LLDB 调试基础

### 编译选项

```bash
g++ -g -std=c++17 -O0 source.cpp -o program
# -g: 包含调试信息
# -O0: 禁用优化，保证调试信息准确
```

### 常用命令

| GDB 命令 | 说明 |
|---------|------|
| `break main` | 在 main 设置断点 |
| `run` | 运行程序 |
| `next / n` | 单步执行（不进入函数） |
| `step / s` | 单步执行（进入函数） |
| `print var` | 打印变量 |
| `info locals` | 打印局部变量 |
| `backtrace / bt` | 打印调用栈 |
| `continue / c` | 继续执行 |
| `watch var` | 监视变量变化 |
| `set var x = 10` | 修改变量值 |
| `break 10 if i == 5` | 条件断点 |

## 5. 日志系统

### 概念讲解

```cpp
enum class LogLevel { DEBUG, INFO, WARN, ERROR, FATAL };

class Logger {
public:
    static Logger& instance() {
        static Logger instance;
        return instance;
    }

    void log(LogLevel level, const std::string& msg) {
        if (level < min_level_) return;
        std::lock_guard<std::mutex> lock(mutex_);
        std::cout << "[" << level_to_string(level) << "] " << msg << std::endl;
    }

    void set_level(LogLevel level) { min_level_ = level; }
    void info(const std::string& msg) { log(LogLevel::INFO, msg); }

private:
    Logger() : min_level_(LogLevel::DEBUG) { }
    std::string level_to_string(LogLevel level) {
        switch (level) {
            case LogLevel::DEBUG: return "DEBUG";
            case LogLevel::INFO:  return "INFO ";
            case LogLevel::WARN:  return "WARN ";
            case LogLevel::ERROR: return "ERROR";
            case LogLevel::FATAL: return "FATAL";
        }
        return "UNKNOWN";
    }
    LogLevel min_level_;
    std::mutex mutex_;
};
```

### 日志级别

```
DEBUG < INFO < WARN < ERROR < FATAL

生产环境：INFO 级别
调试环境：DEBUG 级别
```

## 6. 常见调试技巧

### 断言

```cpp
#include <cassert>

int val = 42;
assert(val > 0 && "值必须为正数");  // 调试断言

// NDEBUG 宏禁用 assert
// #define NDEBUG
// #include <cassert>  // assert 变成空操作

// C++11 static_assert（编译期断言）
static_assert(sizeof(int) >= 4, "int 至少 4 字节");
static_assert(sizeof(void*) == 8, "仅支持 64 位系统");
```

### 内存错误检测工具

| 工具 | 检测内容 |
|------|---------|
| AddressSanitizer (ASan) | 越界访问、use-after-free |
| Valgrind | 内存泄漏、未初始化内存 |
| ThreadSanitizer | 数据竞争 |
| UBSanitizer | 未定义行为 |

```bash
# AddressSanitizer
g++ -fsanitize=address -g source.cpp -o program

# Valgrind
valgrind --leak-check=full ./program
```

## 本章小结

| 概念 | 核心要点 |
|------|---------|
| throw | 抛出异常对象 |
| try/catch | 捕获和处理异常，按类型匹配 |
| 栈展开 | 异常时逐层退出，调用析构函数 |
| RAII | 析构函数保证资源释放，配合异常安全 |
| noexcept | 声明函数不抛异常，跳过栈展开 |
| 异常层次 | std::exception → logic_error / runtime_error |
| 断言 | assert（运行时）、static_assert（编译时） |
| 调试工具 | GDB/LLDB、ASan、Valgrind |
