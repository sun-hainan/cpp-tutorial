## 1. Socket 与 TCP/UDP

### 生活类比

> **socket** 就像一个**电话插座**——IP 是电话号码，Port 是分机号，两者组合就是唯一的"号码牌"。
> **TCP** 像**打电话**——对方接通了才能说话，说话顺序固定，有确认机制。
> **UDP** 像**发短信**——发出去就不管了，不保证对方收到。

### 概念讲解

| 特性 | TCP | UDP |
|------|-----|-----|
| 连接 | 面向连接 | 无连接 |
| 可靠性 | 可靠（有重传/确认） | 不可靠 |
| 顺序 | 按序到达 | 不保证顺序 |
| 速度 | 较慢 | 快 |
| 头部开销 | 20-60 字节 | 8 字节 |
| 适用场景 | 文件传输、网页 | 视频、语音、DNS |

### 关键概念

```cpp
// 字节序转换
htons() / htonl()  // 主机→网络（大端）
ntohs() / ntohl()  // 网络→主机

// sockaddr_in 结构
struct sockaddr_in {
    short sin_family;       // AF_INET（IPv4）
    unsigned short sin_port; // 端口号（网络字节序）
    struct in_addr sin_addr; // IP 地址
};
```

## 2. TCP 服务器与客户端

### 流程对比

```
TCP 服务器：                    TCP 客户端：
    │                              │
    ▼                              │
socket()  创建 socket              │
    │                              │
bind()    绑定地址                  │
    │                              │
listen()  监听连接         ───────►│
    │                              │
accept()  接受连接（新 socket）     │
    │                              │
recv()    接收数据          ◄──────│
    │                         connect()
send()    发送数据          ──────►│
    │                              │
closesocket() 关闭连接              │
```

### 代码示例

```cpp
#ifdef _WIN32
    // Windows Winsock 初始化
    WSADATA wsa;
    WSAStartup(MAKEWORD(2, 2), &wsa);
#endif

// 创建 socket
SOCKET server_sock = socket(AF_INET, SOCK_STREAM, IPPROTO_TCP);

// 绑定地址
sockaddr_in addr;
addr.sin_family = AF_INET;
addr.sin_port = htons(8888);
addr.sin_addr.s_addr = INADDR_ANY;
bind(server_sock, (sockaddr*)&addr, sizeof(addr));

// 监听
listen(server_sock, 5);

// 接受连接
SOCKET client = accept(server_sock, nullptr, nullptr);

// 收发数据
// char buffer[1024];
// recv(client, buffer, sizeof(buffer), 0);
// send(client, response, strlen(response), 0);

// 清理
closesocket(server_sock);
#ifdef _WIN32
    WSACleanup();
#endif
```

## 3. select 与 epoll

### 概念对比

| 特性 | select | epoll |
|------|--------|-------|
| 最大 fd 数 | FD_SETSIZE (1024) | 无限制 |
| 时间复杂度 | O(n) | O(1) |
| 触发方式 | 水平触发 | 水平 + 边缘触发 |
| 内核数据结构 | 每次调用传入 | 复用 |
| 跨平台 | 跨平台 | 仅 Linux |

### 代码示例

```cpp
// select 用法
fd_set readfds;
FD_ZERO(&readfds);
FD_SET(socket_fd, &readfds);
struct timeval timeout = {1, 0};
int n = select(max_fd + 1, &readfds, nullptr, nullptr, &timeout);
if (FD_ISSET(socket_fd, &readfds)) {
    // socket 可读
}

// int epfd = epoll_create1(0);
// struct epoll_event ev = { .events = EPOLLIN, .data.fd = sock_fd };
// epoll_ctl(epfd, EPOLL_CTL_ADD, sock_fd, &ev);
// struct epoll_event events[10];
// int n = epoll_wait(epfd, events, 10, -1);
```

## 4. UDP 编程

### 生活类比

> **UDP** 就像**对讲机**——按下就说，不需要对方确认收到，频道里喊一声大家都能听到。

### 代码示例

```cpp
// 创建 UDP socket
SOCKET udp_sock = socket(AF_INET, SOCK_DGRAM, IPPROTO_UDP);

// 绑定（服务器）
bind(udp_sock, (sockaddr*)&addr, sizeof(addr));

// 发送（客户端）

// socklen_t addr_len = sizeof(src_addr);

## 5. HTTP 协议基本结构

### 请求格式

```
GET /index.html HTTP/1.1\r\n
Host: www.example.com\r\n
User-Agent: C++ Client\r\n
\r\n
[可选 body]
```

```

Content-Length: 123\r\n
\r\n

## 7. 网络安全考虑

### 常见问题

| 问题 | 解决方案 |
|------|---------|
| 字节序 | 始终用 htons/ntohs 处理端口 |
| Socket 泄漏 | RAII 封装 SOCKET |
| 缓冲区溢出 | 检查 recv 返回值，验证输入大小 |
| 整数溢出 | Content-Length 验证 |
| DNS 欺骗 | 验证证书（HTTPS） |
| 数据竞争 | 多线程访问 socket 加锁 |

### 安全建议

```
✅ 优先使用 HTTPS
✅ 限制连接数/频率（防止 DoS）
✅ 输入验证（所有外部数据不可信）
✅ 使用成熟库（libcurl/Asio）
```

## 本章小结

| 概念 | 核心要点 |
|------|---------|
| Socket | IP + Port，网络通信端点 |
| TCP | 面向连接，可靠传输，文件传输/HTTP |
| UDP | 无连接，快速，DNS/视频/语音 |
| select | O(n) 有 fd 限制，跨平台 |
| epoll | O(1) 无限制，Linux 独有 |
| HTTP | 文本协议，请求/响应格式清晰 |
| Asio | 跨平台异步 I/O 库 |
| 安全 | 验证输入，使用 TLS，限制资源 |
