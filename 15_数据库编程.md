## 1. SQL 基础

### 生活类比

> **SQL** 就像一个**图书馆管理员**——你告诉它"我要找所有 1980 年以后出版的历史书，按作者排序"，它帮你找好，而不是你自己去书架上一本本翻。

### 四大基本操作（CRUD）

```sql
-- INSERT：插入
INSERT INTO users (name, age, email)
VALUES ('Alice', 25, 'alice@example.com');

-- SELECT：查询
SELECT id, name, age FROM users
WHERE age > 18 ORDER BY age DESC LIMIT 10;

-- UPDATE：更新
UPDATE users SET age = 26 WHERE name = 'Alice';

-- DELETE：删除
DELETE FROM users WHERE id = 5;
```

### 常用子句

| 子句 | 说明 |
|------|------|
| WHERE | 条件过滤 |
| ORDER BY | 排序 |
| LIMIT | 限制数量 |
| JOIN | 表连接 |
| GROUP BY | 分组 |
| DISTINCT | 去重 |

## 2. 参数化查询与 SQL 注入防护

### SQL 注入原理

```sql
-- 不安全的查询（字符串拼接）
-- SELECT * FROM users WHERE name = '" + user_input + "'
-- 如果 user_input = "' OR '1'='1"
-- 变成：SELECT * FROM users WHERE name = '' OR '1'='1'
-- 结果：整张表被暴露！

-- ✅ 安全：参数化查询
PREPARE stmt FROM 'SELECT * FROM users WHERE name = ?';
EXECUTE stmt USING @name;
```

## 3. 事务与 ACID

### ACID 特性

| 特性 | 说明 |
|------|------|
| Atomicity | 原子性：全部成功或全部失败 |
| Consistency | 一致性：事务前后数据库状态一致 |
| Isolation | 隔离性：并发事务互不干扰 |
| Durability | 持久性：提交后数据永久保存 |

### 事务控制

```sql
START TRANSACTION;
  UPDATE accounts SET balance = balance - 100 WHERE id = 1;
  UPDATE accounts SET balance = balance + 100 WHERE id = 2;
COMMIT;
-- 或
ROLLBACK;  -- 出错时回滚
```

### 隔离级别

| 级别 | 脏读 | 不可重复读 | 幻读 |
|------|------|-----------|------|
| Read Uncommitted | 可能 | 可能 | 可能 |
| Read Committed | 不可能 | 可能 | 可能 |
| Repeatable Read | 不可能 | 不可能 | 可能 |
| Serializable | 不可能 | 不可能 | 不可能 |

## 5. ODBC

### 基本流程

```
1. SQLAllocHandle - 分配环境句柄
2. SQLSetEnvAttr - 设置环境属性
3. SQLAllocHandle - 分配连接句柄
4. SQLConnect - 连接数据库
5. SQLExecDirect - 执行 SQL
6. SQLFetch / SQLGetData - 获取结果
7. SQLFreeHandle - 释放资源
```

### ODBC vs Connector/C++

| 特性 | ODBC | Connector/C++ |
|------|------|-------------|
| 通用性 | 跨数据库 | MySQL 专用 |
| 易用性 | 较底层 | 面向对象 |
| 性能 | 高 | 高 |

## 6. ORM

### ORM 概念

> **ORM**（Object-Relational Mapping）就像一个**翻译官**——把数据库表翻译成编程语言中的对象，让你可以用"面向对象"的方式操作数据库。

```cpp
// ORM 概念示例
// class User {
// public:
//     int id;
//     std::string name;
//     int age;
//     std::string email;
// };
// db.load(u, 1);  // SELECT * FROM users WHERE id = 1
// u.age = 26;
// db.store(u);    // UPDATE users SET age = 26 WHERE id = 1
```

### ORM vs 手写 SQL

| 方面 | ORM | 手写 SQL |
|------|-----|---------|
| 开发效率 | 高 | 低 |
| 性能 | 略低 | 高 |
| 复杂查询 | 困难 | 灵活 |
| 移植性 | 好 | 差 |

## 7. 连接池与性能优化

### 连接池原理

```
启动时创建 N 个连接
       │
       ▼
  ┌──────────┐
  │ 连接池     │
  └──────────┘
       │
   请求时取
   可用连接
       │
       ▼
   使用完毕
   归还连接
```

### 性能优化技巧

```
1. 使用连接池
2. 批量插入（INSERT ... VALUES (), (), ...）
3. 预编译语句复用
4. 合理使用索引
5. 分页查询代替大结果集
6. 异步查询
```
