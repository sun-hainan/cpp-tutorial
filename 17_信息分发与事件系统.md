## 1. 观察者模式

### 生活类比

> **观察者模式**就像**微信公众号订阅**——你关注了公众号，每当有新文章发布（事件），所有关注者都会收到推送。

### 概念讲解

```cpp
class EventBus {
public:
    void subscribe(const std::string& event_type, IObserver* observer) {
        observers_[event_type].push_back(observer);
    }

    void unsubscribe(const std::string& event_type, IObserver* observer) {
        auto& list = observers_[event_type];
        list.erase(std::remove(list.begin(), list.end(), observer), list.end());
    }

    void publish(const std::string& event_type, const std::string& data) {
        auto it = observers_.find(event_type);
        if (it != observers_.end()) {
            for (auto* obs : it->second) {
                obs->on_event(event_type, data);
            }
        }
    }

private:
    std::map<std::string, std::vector<IObserver*>> observers_;
};
```

## 2. 信号槽机制

### 生活类比

> **信号槽**就像**电视遥控器**——按下按钮（信号），电视执行对应操作（槽函数）。可以随时连接/断开。

### 概念讲解

```cpp
class Signal {
public:
    using Slot = std::function<void()>;

    int connect(Slot slot) {
        int id = next_id_++;
        slots_[id] = std::move(slot);
        return id;
    }

    void disconnect(int id) {
        slots_.erase(id);
    }

    void emit() {
        for (auto& [id, slot] : slots_) {
            slot();
        }
    }

private:
    int next_id_ = 0;
    std::map<int, Slot> slots_;
};

// 带类型的信号槽
template<typename... Args>
class TypedSignal {
public:
    using Handler = std::function<void(Args...)>;

    int connect(Handler handler) {
        int id = next_id_++;
        handlers_[id] = std::move(handler);
        return id;
    }

    void emit(Args... args) {
        for (auto& [id, h] : handlers_) {
            h(args...);
        }
    }

private:
    int next_id_ = 0;
    std::map<int, Handler> handlers_;
};
```

## 3. 消息队列

### 生活类比

> **消息队列**就像**快递柜**——寄件人把包裹放进去，收件人随时可以去取。不需要双方同时在线。

### 概念讲解

```cpp
template<typename T>
class MessageQueue {
public:
    void push(T msg) {
        std::lock_guard<std::mutex> lock(mutex_);
        queue_.push(std::move(msg));
    }

    bool pop(T& msg) {
        std::lock_guard<std::mutex> lock(mutex_);
        if (queue_.empty()) return false;
        msg = std::move(queue_.front());
        queue_.pop();
        return true;
    }

private:
    mutable std::mutex mutex_;
    std::queue<T> queue_;
};

// 消息结构
struct Message {
    std::string type;
    std::variant<int, std::string> payload;
};
```

## 4. 事件循环

### 生活类比

> **事件循环**就像**餐厅的服务员**——不停巡逻看有没有客人招手（事件），有就过去服务，然后继续巡逻。

### 概念讲解

```cpp
class EventLoop {
public:
    using Task = std::function<void()>;

    void post(Task task) {
        std::lock_guard<std::mutex> lock(mutex_);
        tasks_.push(std::move(task));
    }

    void run() {
        running_ = true;
        while (running_) {
            std::vector<Task> local_tasks;
            {
                std::lock_guard<std::mutex> lock(mutex_);
                local_tasks.swap(tasks_);
            }
            for (auto& task : local_tasks) {
                if (task) task();
            }
            std::this_thread::sleep_for(std::chrono::milliseconds(10));
        }
    }

    void stop() { running_ = false; }

private:
    mutable std::mutex mutex_;
    std::vector<Task> tasks_;
    std::atomic<bool> running_{false};
};
```

## 5. 中间件模式

### 生活类比

> **中间件**就像**酒店的前台**——客人入住要经过多个前台：登记、查身份、付押金、领房卡。每一层只做自己的事，然后传给下一层。

### 概念讲解

```cpp
class MiddlewareChain {
public:
    using Handler = std::function<void(int)>;

    void use(const std::string& name, Handler handler) {
        middlewares_.push_back({name, std::move(handler)});
    }

    void execute(int request) {
        int index = 0;
        std::function<void(int)> next = [&](int req) {
            if (index < static_cast<int>(middlewares_.size())) {
                auto& mw = middlewares_[index++];
                std::cout << "进入中间件: " << mw.name << std::endl;
                mw.handler(req);
            } else {
                std::cout << "最终处理请求: " << req << std::endl;
            }
        };
        next(request);
    }

private:
    struct Middleware {
        std::string name;
        Handler handler;
    };
    std::vector<Middleware> middlewares_;
};
```

## 6. 回调与异步模式

### 生活类比

> **回调**就像**叫外卖**——你下单时留了电话号码（回调函数），外卖到了商家打电话叫你（触发回调）。

### 概念讲解

```cpp
using Callback = std::function<void(int, const std::string&)>;

auto async_operation = [](Callback cb) {
    std::thread t([cb]() {
        std::this_thread::sleep_for(std::chrono::milliseconds(100));
        cb(200, "Operation completed");
    });
    t.detach();
};

async_operation([](int code, const std::string& msg) {
    std::cout << "回调收到: code=" << code << ", msg=" << msg << std::endl;
});

// Promise/Future 模式
std::cout << "Future/Promise 替代回调，避免回调地狱" << std::endl;
```

## 7. Qt 信号槽 vs 自定义事件系统

### 对比

| 特性 | 自定义 | Qt |
|------|--------|-----|
| 依赖 | 仅 STL | Qt 框架 |
| 线程安全 | 需自己实现 | 内置 |
| 跨线程 | 需要手动处理 | 自动 QueuedConnection |
| 内存管理 | 需手动管理 | parent/child |
| 性能 | 可能更快 | 略开销 |

## 本章小结

| 概念 | 核心要点 |
|------|---------|
| 观察者模式 | 一对多依赖，EventBus 是典型实现 |
| 信号槽 | 安全的回调容器，支持连接/断开 |
| 消息队列 | 异步通信，解耦生产者和消费者 |
| 事件循环 | 单线程异步处理，GUI 和网络服务器常用 |
| 中间件 | 链式处理，职责分离，便于组合 |
| 回调 | 异步基础，Promise/async 更清晰 |
| Qt 信号槽 | 完整的事件系统，适合大型项目 |
