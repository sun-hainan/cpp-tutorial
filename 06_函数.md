## 1. 函数基础：代码的"功能模块"

### 生活类比

> 函数就像厨房里的**食谱**——有输入（食材）、有处理步骤（烹饪方法）、有输出（成品菜）。调用函数就是按照食谱做菜。

### 概念讲解

**函数签名** = 函数名 + 参数类型列表（**不含**返回类型）

```cpp
// 函数签名：max(int, int)
int max(int a, int b) {
    return a > b ? a : b;
}

// 多返回值（C++17 结构化绑定）
auto divide = [](int a, int b) -> std::pair<int, int> {
    return {a / b, a % b};
};
auto [quotient, remainder] = divide(17, 5);  // C++17 结构化绑定

// 函数作为参数（函数指针）
auto apply = [](int x, int y, int (*op)(int, int)) -> int {
    return op(x, y);
};
```

## 2. 值传递、引用传递、指针传递

### 生活类比

> - **值传递**：把文件的**复印件**给对方，原件不动
> - **引用传递**：给一把**备用钥匙**，对方能改你的锁
> - **指针传递**：告诉对方**门牌号**，对方可以去找房子

### 概念讲解

| 传递方式 | 语法 | 效果 | 适用场景 |
|---------|------|------|---------|
| 值传递 | `func(T val)` | 传递副本 | 内置类型（小对象） |
| 引用传递 | `func(T& ref)` | 传递别名，可修改 | 需要修改或大对象 |
| 指针传递 | `func(T* ptr)` | 传递地址，可解引用修改 | C 风格接口、可空参数 |
| const 引用 | `func(const T& ref)` | 只读不修改 | 大对象只读 |

### 代码示例

```cpp
int a = 10, b = 20;

// 值传递：副本，不影响原变量
auto by_value = [](int x) { x = 999; return x; };
by_value(a);  // a 仍然是 10

// 引用传递：别名，修改影响原变量
auto by_reference = [](int& x) { x = 999; return x; };
by_reference(b);  // b 变成 999

// 指针传递：地址，解引用可修改
auto by_pointer = [](int* px) {
    if (px) *px = 888;  // 需要检查空指针
    return px ? *px : 0;
};
by_pointer(&a);  // a 变成 888

// const 引用：只读
auto const_ref = [](const std::string& s) -> size_t {
    // s += " modified";  // ❌ 编译错误
    return s.length();
};
```

### 最佳实践

```
C++ 参数传递最佳实践：
  ✅ 内置类型（int/double）→ 传值或 const 引用
  ✅ 大对象（string/vector）→ const 引用或移动语义
  ✅ 需要修改时 → 传引用（&）
  ✅ 可空参数 → 传指针
```

## 3. 函数重载与重写

### 生活类比

> **函数重载**就像同名的**不同工种师傅**——"张师傅"可以是木匠张师傅、铁匠张师傅、水电工张师傅，根据你给的材料（参数）决定叫哪位。
> **函数重写**就像**徒弟学师傅的手艺**——徒弟的同名方法覆盖了师傅的方法。

### 概念讲解

| 概念 | 重载（Overload） | 重写（Override） |
|------|-----------------|------------------|
| 作用域 | 同一作用域 | 派生类作用域 |
| 函数名 | 相同 | 相同 |
| 参数 | 必须不同 | 必须相同 |
| virtual | 不需要 | **必须**有 |
| 绑定时机 | 编译时 | 运行时 |

```cpp
// 函数重载（同一作用域，参数不同）
auto max = [](int a, int b) -> int { return a > b ? a : b; };
auto max = [](double a, double b) -> double { return a > b ? a : b; };
auto max = [](int a, int b, int c) -> int { return max(a, b) > c ? max(a, b) : c; };

// ❌ 不能仅通过返回类型重载！
// auto func() -> int { return 1; }
// auto func() -> double { return 1.0; }  // 冲突！

// 函数重写（派生类覆盖基类虚函数）
struct Base {
    virtual void foo() const;
};
struct Derived : Base {
    void foo() const override;  // C++11 override 关键字
};
```

## 4. 递归：函数调用自己

### 生活类比

> 递归就像**俄罗斯套娃**——打开一个娃娃，里面还有更小的同款娃娃，直到打开最小的那个（基准情况）才开始往回装。

### 概念讲解

**递归三要素**：
1. **基准情况**：最小问题，直接返回
2. **递归调用**：调用自己解决更小的问题
3. **返回值**：合并子问题结果

```cpp
// 阶乘
auto factorial = [](auto&& self, int n) -> long long {
    if (n <= 1) return 1;                  // 基准情况
    return n * self(self, n - 1);            // 递归调用
};

// 斐波那契数列
auto fib = [](auto&& fib, int n) -> long long {
    if (n <= 1) return n;                   // 基准情况
    return fib(fib, n - 1) + fib(fib, n - 2);  // 递归调用
};

// 记忆化递归（避免重复计算）
auto fib_memo = [](int n) -> long long {
    static std::vector<long long> memo(101, -1);
    if (memo[n] != -1) return memo[n];       // 命中缓存
    if (n <= 1) return memo[n] = n;
    return memo[n] = fib_memo(n - 1) + fib_memo(n - 2);
};

// 尾递归（编译器可优化为循环）
auto factorial_tail = [](int n, long long acc = 1) -> long long {
    if (n <= 1) return acc;
    return factorial_tail(n - 1, n * acc);
};
```

## 5. 可变参数模板

### 生活类比

> 可变参数模板就像**万能调料盒**——可以放任意数量的调料，根据放进去的东西自动适应。

### 概念讲解

```cpp
// C++11 可变参数模板
auto sum = [](auto... args) {
    return (args + ...);  // C++17 折叠表达式
};
std::cout << sum(1, 2, 3, 4, 5) << std::endl;  // 15

// 打印任意数量参数
auto print_all = [](auto... args) {
    ((std::cout << args << " "), ...);  // C++17 折叠表达式
};
print_all("Hello", 42, 3.14, 'A');

// C++11 std::initializer_list（更安全的可变参数）
auto sum_init_list = [](std::initializer_list<int> vals) -> int {
    int sum = 0;
    for (int v : vals) sum += v;
    return sum;
};
sum_init_list({1, 2, 3, 4, 5});
```

## 6. lambda 表达式：匿名函数对象

### 生活类比

> lambda 就像**一次性贴纸**——你可以随手写一个临时的小功能，贴在需要的地方，用完就扔，不需要正式命名。

### 概念讲解

**lambda 语法**：
```
[capture] (params) -> ret_type { body }
```

**捕获列表**：

| 捕获方式 | 含义 |
|---------|------|
| `[]` | 不捕获 |
| `[=]` | 值捕获（拷贝） |
| `[&]` | 引用捕获 |
| `[x]` | 只捕获 x（值） |
| `[&x]` | 只捕获 x（引用） |
| `[=, &x]` | 默认值捕获，x 引用捕获 |
| `[&, x]` | 默认引用捕获，x 值捕获 |

### 代码示例

```cpp
int x = 10;

// 值捕获（创建时拷贝）
auto captured_value = [x] { return x; };

// 引用捕获（引用原变量，注意生命周期）
auto captured_ref = [&x] { return x; };

// C++14 泛型 lambda
auto generic = [](auto a, auto b) { return a + b; };

// mutable lambda（可修改值捕获的副本）
int y = 5;
auto mutable_lambda = [y]() mutable {
    y = 10;  // 修改副本，不影响外部 y
    return y;
};

// C++17 初始化捕获（移动语义）
std::vector<int> vec = {1, 2, 3};
auto moved = [v = std::move(vec)]() mutable { return v.size(); };

// std::function 包装 lambda
std::function<int(int, int)> fn = [](int a, int b) { return a * b; };
```

## 7. 闭包：带状态的 lambda

### 生活类比喻

> 闭包就像一个**带记忆的手提包**——不只是装东西，还有记忆功能，能记住上次用的时候的状态。

### 概念讲解

**闭包** = lambda + 捕获的外部变量

```cpp
// 计数器闭包（函数工厂）
auto make_counter = []() {
    int count = 0;  // 局部状态
    return [count]() mutable {  // 注意：count 是拷贝
        ++count;
        return count;
    };
};

auto counter1 = make_counter();
counter1();  // 1
counter1();  // 2
counter1();  // 3

// 安全的闭包（用 shared_ptr）
auto make_counter_safe = []() {
    auto count = std::make_shared<int>(0);
    return [count]() mutable {
        ++(*count);
        return *count;
    };
};

auto counter3 = make_counter_safe();
counter3();  // 1
counter3();  // 2
```

### ⚠️ 警告：悬挂引用

```cpp
// ❌ 危险！count 在函数返回后销毁
auto make_counter_bad = []() {
    int count = 0;
    return [&count]() -> int& {  // 返回引用
        return ++count;
    };
};
```

## 本章小结

| 概念 | 核心要点 |
|------|---------|
| 函数签名 | 函数名 + 参数类型，不含返回类型 |
| 值传递 | 传递副本，不修改原变量 |
| 引用传递 | 传递别名，可修改原变量，const 引用只读 |
| 指针传递 | 显式传递地址，需检查空指针 |
| 重载 | 同作用域同名不同参，编译时绑定 |
| 重写 | 派生类覆盖基类虚函数，运行时绑定 |
| 递归 | 三要素：基准情况、递归调用、返回值 |
| 尾递归 | 可被编译器优化为循环 |
| 可变参数 | C++11 模板参数包，C++17 折叠表达式 |
| lambda | 匿名函数对象，[捕获] (参数) -> 返回 { body } |
| 闭包 | lambda + 捕获的外部变量，注意生命周期 |
