## 1. RTTI 与 typeid

### 生活类比

> **RTTI**（运行时类型信息）就像人的**身份证**——可以随时查询一个对象"到底是人还是狗"。

### 概念讲解

```cpp
class Base {
public:
    virtual ~Base() = default;  // 有虚函数才有 RTTI
    virtual void foo() { std::cout << "Base::foo" << std::endl; }
};

class Derived : public Base {
public:
    void foo() override { std::cout << "Derived::foo" << std::endl; }
    void derived_only() { std::cout << "Derived::derived_only" << std::endl; }
};

Base* b1 = new Base();
Base* b2 = new Derived();

// typeid：获取类型信息
std::cout << typeid(*b1).name() << std::endl;  // 类型名
std::cout << typeid(*b2).name() << std::endl;  // 可能是不同类型

// 判断类型是否相同
if (typeid(*b1) == typeid(*b2)) {
    std::cout << "类型相同" << std::endl;
} else {
    std::cout << "类型不同" << std::endl;
}
```

### 重要前提

> **必须有虚函数**才能使用 RTTI！无虚函数的类没有 type_info。

## 2. dynamic_cast：安全的类型转换

### 生活类比

> **dynamic_cast** 就像一个**万能转换器**——尝试把一个东西转成另一个类型，如果不能转就返回 nullptr。

### 概念讲解

```cpp
Base* base_ptr = new Derived();

// 向下转型（Downcast）
Derived* d = dynamic_cast<Derived*>(base_ptr);
if (d) {
    std::cout << "向下转型成功" << std::endl;
    d->derived_only();
} else {
    std::cout << "转型失败（不是 Derived 类型）" << std::endl;
}

// 不安全的向下转型
Base* fake_base = new Base();
Derived* bad = dynamic_cast<Derived*>(fake_base);
if (bad == nullptr) {  // 安全检查！
    std::cout << "安全的空指针检查" << std::endl;
}

// 引用转型（失败时抛出 std::bad_cast）
try {
    Derived& d_ref = dynamic_cast<Derived&>(*base_ptr);
} catch (const std::bad_cast& e) {
    std::cout << "bad_cast 异常" << std::endl;
}
```

## 3. type_traits：编译期类型检测

### 概念讲解

```cpp
// 基本类型查询
std::cout << std::is_integral<int>::value << std::endl;  // 1
std::cout << std::is_floating_point<double>::value << std::endl;  // 1

// C++14 变量模板（_v 后缀）
std::cout << std::is_integral_v<int> << std::endl;  // 1

// 类型关系
std::cout << std::is_same<int, int32_t>::value << std::endl;

// enable_if：条件启用函数模板
template<typename T,
         typename = std::enable_if_t<std::is_integral_v<T>>>
T double_val(T v) { return v * 2; }

// C++17 if constexpr 实现分支
auto process = [](auto val) {
    if constexpr (std::is_integral_v<decltype(val)>) {
        return val * 2;
    } else if constexpr (std::is_floating_point_v<decltype(val)>) {
        return val * 2.0;
    } else {
        return 0;
    }
};
```

## 4. 模板元编程（TMP）

### 生活类比

> **模板元编程**就像一个**会计算的复印机**——把数字输进去，复印机在复印的过程中就帮你算好了，而不是复印完再手算。

### 概念讲解

```cpp
// 编译期阶乘
template<int N>
struct Factorial {
    static const int value = N * Factorial<N - 1>::value;
};

template<>
struct Factorial<0> {
    static const int value = 1;
};

static_assert(Factorial<5>::value == 120);

// 编译期斐波那契
template<int N>
struct Fibonacci {
    static const int value = Fibonacci<N - 1>::value + Fibonacci<N - 2>::value;
};

template<>
struct Fibonacci<0> { static const int value = 0; };
template<>
struct Fibonacci<1> { static const int value = 1; };

// C++14 constexpr 更简洁
constexpr int fib14(int n) {
    if (n <= 1) return n;
    return fib14(n - 1) + fib14(n - 2);
}
```

## 5. constexpr if 与 SFINAE

### 概念讲解

```cpp
// SFINAE：Substitution Failure Is Not An Error
template<typename T>
void q5_sfinae_impl(T* ptr, std::true_type) {
    std::cout << "T 是指针类型" << std::endl;
}

template<typename T>
void q5_sfinae_impl(T, std::false_type) {
    std::cout << "T 不是指针类型" << std::endl;
}

template<typename T>
void q5_sfinae(T val) {
    q5_sfinae_impl(&val, std::is_pointer<T>{});
}

// C++17 constexpr if：更简洁
auto process_v2 = [](auto val) {
    if constexpr (std::is_pointer_v<decltype(val)>) {
        return *val;
    } else if constexpr (std::is_integral_v<decltype(val)>) {
        return val;
    } else {
        return decltype(val){};
    }
};
```

## 6. C++20 Concepts

### 概念讲解

```cpp
// C++17 等效写法
template<typename T,
         typename = std::enable_if_t<std::is_integral_v<T>>>
T add_17(T a, T b) { return a + b; }

// C++20 概念（更清晰）
// template<std::integral T>
// T add(T a, T b) { return a + b; }

// requires 子句
// template<typename T>
//     requires std::integral<T>
// T square(T x) { return x * x; }
```

## 7. 宏展开：预处理器的"文本替换"

### 概念讲解

```cpp
// 好宏：加括号
#define MAX(a, b) ((a) > (b) ? (a) : (b))

// 坏宏：可能出问题
#define BAD_MAX(a, b) (a > b ? a : b)

// 多语句宏：用 do { } while(0) 包裹
#define SAFE_MACRO(x) \
    do { \
        int temp = (x); \
        std::cout << "temp = " << temp << std::endl; \
    } while(0)

// ⚠️ 宏的副作用陷阱
int a = 3, b = 5;
// MAX(++a, b) 展开为 ((++a) > (b) ? (++a) : (b))
// a 被递增两次！结果不可预测！
```

## 本章小结

| 概念 | 核心要点 |
|------|---------|
| RTTI | 依赖虚函数表，typeid 返回类型信息 |
| dynamic_cast | 运行时类型转换，失败返回 nullptr |
| type_traits | 编译期类型查询，enable_if SFINAE |
| 模板元编程 | 编译期计算，递归+特化 |
| constexpr if | C++17 编译期分支，消除 SFINAE 复杂性 |
| Concepts | C++20 清晰的模板约束 |
| 宏 | 文本替换，无类型检查，谨慎使用 |
