## 1. 变量与内存：数据的"家"

### 生活类比

> 变量就像一个**有门牌号的储物箱**：门牌号是地址，里面存的是值。你告诉编译器"我要一个叫 age 的箱子"，编译器就在内存里找个空位给它。

### 概念讲解

**变量 = 名称 + 类型 + 地址 + 值**

内存三大区域：

| 区域 | 存储内容 | 特点 |
|------|---------|------|
| 栈（Stack） | 局部变量 | 编译器自动分配/释放，速度快 |
| 堆（Heap） | 动态分配 | 手动管理（new/delete）或智能指针 |
| 全局/静态区 | 全局变量、static | 程序生命周期内存在 |

```cpp
int age = 25;  // 栈上分配 4 字节（通常）
std::cout << "age 地址 = " << &age << ", 值 = " << age << std::endl;
```

### 内存布局图

```
内存地址（由低到高）

┌─────────────────────────────────────┐
│  代码区（Code）     │  存放程序指令    │
├─────────────────────────────────────┤
│  全局/静态区          │  全局变量、static │
├─────────────────────────────────────┤
│  堆（Heap）          │  new 分配 ↑      │
│                     │         ↓ 释放    │
├─────────────────────────────────────┤
│  栈（Stack）         │  局部变量 ↓      │
│                     │  函数调用 ↑      │
└─────────────────────────────────────┘
```

## 2. 基本数据类型：不同规格的"容器"

### 生活类比

> 就像容器有不同规格（碗、杯子、箱子），C++ 的数据类型也有不同"容量"——装整数的、装小数的、装字符的。

### 概念讲解

| 类型 | 关键字 | 典型大小 | 典型范围 |
|------|--------|---------|---------|
| 布尔 | bool | 1 字节 | true / false |
| 字符 | char | 1 字节 | -128 ~ 127 |
| 整数 | int | 4 字节 | -2³¹ ~ 2³¹-1 |
| 长整数 | long | ≥4 字节 | 平台相关 |
| 超长整数 | long long | 8 字节 | -2⁶³ ~ 2⁶³-1 |
| 单精度浮点 | float | 4 字节 | ±3.4e±38 |
| 双精度浮点 | double | 8 字节 | ±1.7e±308 |

**注意**：`int` 的大小**平台相关**，16 位系统是 2 字节，32 位通常是 4 字节。

### 代码示例

```cpp
bool    is_valid = true;           // 1 字节
char    grade    = 'A';            // 1 字节
int     count    = 100;            // 通常 4 字节
float   price    = 99.99f;         // 4 字节（f 后缀表示 float）
double  salary   = 5000.50;        // 8 字节
long    big_num  = 1'000'000L;     // C++14 数字分隔符
long long very_big = 1e12LL;       // 8 字节

// C++11 固定宽度整数
int8_t  byte_val  = 127;    // 1 字节
int64_t int64_val = 9'000'000'000'000LL;  // 8 字节

// 打印各类型大小
std::cout << "sizeof(int) = " << sizeof(int) << " 字节" << std::endl;
```

## 3. 作用域与生命周期：变量的"时空观"

### 生活类比

> **作用域**像是一个房间——出了房间就看不见里面的东西。**生命周期**像是一段旅程——从出发（创建）到归来（销毁）。

### 概念讲解

```cpp
int a = 10;  // 函数作用域，从此处开始可见

{   // 块作用域开始
    int b = 20;  // 只在这个块内可见
    std::cout << "块内 a=" << a << ", b=" << b << std::endl;
}   // b 在这里被销毁
// std::cout << b;  // ❌ 编译错误：b 已超出作用域
```

| 类型 | 生命周期 | 初始化 |
|------|---------|--------|
| 自动变量（栈） | 函数结束自动销毁 | 默认未初始化 |
| 静态变量 | 程序结束销毁 | 默认零初始化 |
| 动态内存（堆） | 手动管理 | 默认未初始化 |

## 4. 指针 vs 引用：两个"传送门"

### 生活类比

> **指针**是门牌号的**复制品**——你可以拿着它去找任何房子，但门牌号本身可以为空（nullptr）。
> **引用**是房子的**正式别名**——一旦绑定就不能换房子，但一定有房子对应。

### 概念讲解

```cpp
int x = 42;
int* p = &x;   // 指针：存储变量的地址，可以是 nullptr
int& r = x;    // 引用：变量的别名，必须在创建时绑定
```

### 指针 vs 引用对比表

| 特性 | 指针 | 引用 |
|------|------|------|
| 创建时初始化 | 可选（可为 nullptr） | **必须**绑定对象 |
| 重新赋值 | 可修改指向 | **不可**重新绑定 |
| 空值 | nullptr | 无 |
| sizeof | 平台相关（4/8 字节） | 等于原类型大小 |
| 算术运算 | 支持 | 不支持 |

### 代码示例

```cpp
int x = 42;
int* p = &x;        // 指针：存储变量的地址
int& r = x;         // 引用：变量的别名

r = 100;            // 通过引用修改，等同于修改 x
std::cout << x;     // 输出 100

// C++11 nullptr
int* null_ptr = nullptr;  // 推荐用 nullptr 而非 NULL
```

## 5. 类型转换：数据的"翻译"

### 生活类比

> 类型转换就像把中文翻译成英文——有些翻译是自动的（隐式），有些需要译者帮忙（显式）。

### 概念讲解

**C++ 四种类型转换运算符**：

| 转换方式 | 用途 | 安全性 |
|---------|------|--------|
| static_cast | 编译期转换（int↔double, void*↔T*） | 中 |
| const_cast | 移除/添加 const | 低 |
| reinterpret_cast | 位级重新解释（危险） | 最低 |
| dynamic_cast | 运行时多态类型转换（RTTI） | 高 |

```cpp
// 隐式转换（自动）
int i = 42;
double d = i;           // int → double（可能丢失精度）

// 显式转换（C 风格）
double pi = 3.14159;
int truncated = (int)pi;  // 截断小数部分

// C++ 四种 cast
int sc = static_cast<int>(3.14159);    // 编译期检查
// const_cast：去掉 const（不推荐修改 const 变量！）
const int const_val = 100;
int* modifiable = const_cast<int*>(&const_val);
```

## 6. constexpr：编译期的"超能力"

### 生活类比

> 普通计算是"现场做菜"（运行时），constexpr 是"提前做好放冰箱"（编译期）——吃的时候直接端上桌，零等待。

### 概念讲解

```cpp
constexpr int MAX_SIZE = 1000;          // 编译期常量
constexpr int arr[5] = {1, 2, 3, 4, 5}; // 编译期数组大小

// constexpr 函数（C++14 支持多语句）
constexpr int square(int x) { return x * x; }
constexpr int result = square(16);     // 编译期计算：256

// C++17 constexpr lambda
auto lambda = [](int x) constexpr { return x * 2; };
```

**核心优势**：编译期计算出结果，**运行时零成本**。

## 7. typedef 与 using：给类型起"绰号"

### 生活类比

> `typedef` 和 `using` 就像给复杂名字取**简称**——"中华人民共和国"简称"中国"，类型别名让长名字变短。

### 代码示例

```cpp
// typedef：传统写法
typedef unsigned int uint_t;

// using：C++11 风格（更清晰）
using u32 = uint32_t;
using String = std::string;
using IntVec = std::vector<int>;

// using 支持模板别名（typedef 不支持！）
template<typename T>
using Ptr = T*;  // 模板别名

Ptr<int> p_int = &some_int;
```

## 8. 进制与位运算：底层的"二进制思维"

### 概念讲解

```cpp
int dec = 42;        // 十进制（默认）
int hex = 0x2A;      // 十六进制（0x 前缀）
int oct = 052;       // 八进制（C++11 起不建议）
int bin = 0b101010;  // 二进制（C++14 支持 0b 前缀）
```

### 位运算速查

| 运算 | 符号 | 用途 |
|------|------|------|
| 按位与 | `a & b` | 提取特定位 |
| 按位或 | `a \| b` | 设置特定位 |
| 按位异或 | `a ^ b` | 翻转特定位 |
| 按位取反 | `~a` | 全部位取反 |
| 左移 | `a << n` | 乘以 2ⁿ |
| 右移 | `a >> n` | 除以 2ⁿ |

### 代码示例

```cpp
unsigned int flags = 0b00110101;

// 检查第3位是否设置
bool bit3_set = flags & (1 << 3);

// 设置第2位
flags |= (1 << 2);

// 清除第3位
flags &= ~(1 << 3);

// 判断奇偶（不用 %2）
bool is_odd = num & 1;
```

## 本章小结

| 概念 | 核心要点 |
|------|---------|
| 变量 | 具有名称的内存空间，栈上自动分配，堆上手动管理 |
| 基本类型 | bool(1), char(1), int(4), long(≥4), double(8) |
| 作用域 | 变量名可见的区域，出了作用域变量销毁 |
| 生命周期 | 变量存在的时间，栈变量随函数结束销毁 |
| 指针 | 地址变量，可为空，可重新指向 |
| 引用 | 对象别名，必须初始化，不可重新绑定 |
| 类型转换 | 优先用 static_cast，避免 C 风格 cast |
| constexpr | 编译期计算，运行时零开销 |
| typedef/using | 类型别名，推荐 using（支持模板） |
| 位运算 | 底层高效操作，常用于标志位操作 |
