## 1. 注释：代码的"说明书"

### 生活类比

> 想象你在书中重要段落旁边写的**批注**——注释就是写给未来读者（往往是自己）的解释，帮助理解为什么要这样写，而不是机器。

### 概念讲解

C++ 支持两种注释风格：

| 风格 | 语法 | 适用场景 |
|------|------|---------|
| 单行注释 | `// 这里是注释` | 简短说明、解释复杂逻辑 |
| 多行注释 | `/* 多行<br>注释 */` | 长说明、版权声明 |

**核心原理**：注释在**预处理阶段就被完全删除**，不参与编译，不影响运行效率和最终二进制大小。

```cpp
// 单行注释：编译器看到这里直接跳过
/* 多行注释：
   第一行
   第二行 */
```

### 代码示例（逐行注释）

```cpp
#include <iostream>    // ① 包含标准输入输出库（预处理器阶段展开）
#include <string>      // ② C++ 标准字符串类型所需

// ============================================================
// Q1: 为什么 C++ 需要注释？有哪些注释风格？
// ============================================================
void q1_comments() {
    // 单行注释：编译时会被预处理器完全删除，不影响运行效率
    // 推荐用于简短说明、解释复杂逻辑、标记 TODO
    /*
     * 多行注释（块注释）：
     * 适用于较长说明、版权声明、函数接口描述。
     * 注意：块注释不能嵌套！
     */
    std::cout << "Q1: 注释风格演示" << std::endl;
}
```

### ASCII 图示

```
源代码文件（.cpp）
    │
    ▼
┌─────────────────┐
│  预处理阶段       │   ← 注释在这里被删除
│  #include 展开   │
└────────┬────────┘
         ▼
┌─────────────────┐
│  编译阶段        │   ← 纯净的代码被编译
└────────┬────────┘
         ▼
    可执行文件（无注释）
```

### 常见错误

| 错误 | 说明 |
|------|------|
| `/* 块注释嵌套 /* ... */ */` | 块注释**不能嵌套**，第一个 `*/` 就会结束整个块 |
| 中文全角字符在注释中 | 某些老旧编译器可能不支持 UTF-8 注释 |
| 注释过度 | 好的代码应该是**自解释的**，注释解释"为什么"而非"是什么" |

## 2. 缩进规范：代码的"排版"

### 生活类比

> 缩进就像文章的**段落缩进**——同一级别的内容对齐，看起来层次分明。C++ 的缩进是**风格约定**而非语法要求，而 Python 则强制用缩进表示代码块。

### 概念讲解

C++ **不强制**缩进，但主流风格有三种：

| 风格 | 缩进量 | 代表项目 |
|------|--------|---------|
| 4 空格 | `    if (...)` | Linux 内核 |
| 2 空格 | `  if (...)` | Google C++ Style Guide |
| Tab | `→  if (...)` | 争议较大，不推荐 |

**硬件视角**：缩进是**空白字符**，编译器完全忽略它们——它们是纯粹的人类可读性工具。

```cpp
// ✅ 正确缩进示例
if (true) {
    std::cout << "正确缩进：语句在块内缩进一层" << std::endl;
} else {
    std::cout << "else 与 if 对齐" << std::endl;
}
```

### 常见错误

```cpp
// ❌ 滥用 Tab（不同编辑器显示不一致）
// ❌ 混用空格和 Tab（导致代码在某些环境下错位）
// ❌ 过深嵌套（超过 3-4 层应该考虑重构）
```

## 3. 标识符：给变量起名字

### 生活类比

> 标识符就像人的**姓名**——需要遵循一定规则，且应该**见名知意**。变量名 `age` 比 `a` 更易理解。

### 概念讲解

标识符命名规则：

```
规则：
  ✅ 由字母、数字、下划线组成
  ✅ 不能以数字开头
  ✅ 区分大小写（age ≠ Age ≠ AGE）
  ✅ 不能是关键字（int, class, if 等）

命名建议：
  ✅ 蛇形命名（snake_case）：total_count, user_name
  ✅ 驼峰命名（camelCase）：maxValue, totalCount
  ✅ 类名首字母大写：class MyClass
  ❌ 下划线开头的名字（保留给编译器/标准库）
```

### 代码示例（逐行注释）

```cpp
int age = 25;              // ✅ 合法：语义明确
int _private = 1;         // ⚠️ 合法但建议避免：下划线开头是保留的
int total_count = 100;    // ✅ 蛇形命名
int maxValue = 200;       // ✅ 驼峰命名
// int 2ndPlace = 3;      // ❌ 非法：数字开头
// int class = 1;         // ❌ 非法：class 是关键字
```

## 4. 关键字：C++ 的"保留词"

### 生活类比

> 关键字就像**公共设施名称**——"北京站"不能随便给个人用，C++ 的 `if`、`class` 也不能用作变量名。

### 概念讲解

C++ 关键字数量随标准递增：

| C++ 版本 | 新增关键字数 | 代表新增 |
|---------|-------------|---------|
| C++98 | ~60 个 | auto, break, class, if... |
| C++11 | +13 个 | nullptr, constexpr, decltype |
| C++14 | +1 个 | [[deprecated]] |
| C++17 | +10+ 个 | if constexpr, inline variables |
| C++20 | +20+ 个 | concept, requires, co_await |

```cpp
// C++17 constexpr if 示例
constexpr int version = 17;
if constexpr (version >= 17) {
    std::cout << "C++17+ 特性可用" << std::endl;
}
```

## 5. 语句结束符：分号的艺术

### 生活类比

> 分号就像句子的**句号**——告诉编译器"这个句子说完了"。没有句号，句子会连在一起，编译器就懵了。

### 概念讲解

```cpp
int x = 10;     // ✅ 声明语句以分号结尾
x = 20;         // ✅ 赋值语句以分号结尾
;               // ✅ 空语句（合法，但不推荐）

// ❌ 忘记分号
if (true)
    std::cout << "No semicolon"  // 编译错误！
```

**特别注意**：`if`、`for`、`while` 后面**不要加分号**——否则空语句就是它的主体：

```cpp
for (int i = 0; i < 10; i++);  // ⚠️ 空循环体！分号是循环体
    std::cout << i << std::endl;  // 这行只执行一次
```

## 6. 预处理指令：编译前的"准备工作"

### 生活类比

> 预处理就像做菜前的**准备工作**——洗菜、切菜在正式炒菜之前完成。预处理指令在正式编译之前"预处理"源代码。

### 概念讲解

预处理指令以 `#` 开头，在编译前执行：

```cpp
// #include：文件包含
#include <iostream>    // 系统/标准库目录查找
#include "myheader.h"  // 当前目录优先，再查系统目录

// #define：宏定义（文本替换）
#define MAX_SIZE 1024
std::cout << MAX_SIZE << std::endl;  // 预处理后变成 std::cout << 1024 << std::endl;

// 条件编译
#define ENABLE_DEBUG 1
#if ENABLE_DEBUG
    std::cout << "调试模式已开启" << std::endl;
#else
    std::cout << "发布模式" << std::endl;
#endif
```

### 预处理流程图

```
源代码 (.cpp)
    │
    ▼ [预处理阶段]
┌──────────────────────────────────────┐
│  1. #include 文件展开                  │
│  2. #define 宏展开                     │
│  3. 条件编译 (#if/#else)              │
│  4. 删除注释                          │
└──────────────────────────────────────┘
    │
    ▼ [生成 .i 预处理文件]
编译 → 汇编 → 链接 → 可执行文件
```

## 7. C++ 代码基本结构

### 生活类比

> C++ 程序就像一篇**格式严谨的文章**：先写标题（头文件），再写正文（函数定义），最后是落款（main 函数）。

### 概念讲解

标准 C++ 程序结构：

```cpp
// 1. 头文件包含
#include <iostream>
#include <string>

// 2. using 声明（可选）
using namespace std;

// 3. 类定义（可选）
class Widget { /* ... */ };

// 4. 函数定义
void helper() { /* ... */ }

// 5. main 函数（程序入口！）
int main() {
    std::cout << "Hello, World!" << std::endl;
    return 0;  // 0 = 成功，非0 = 错误码
}
```

## 本章小结

| 概念 | 核心要点 |
|------|---------|
| 注释 | 单行 `//` 和多行 `/* */`，预处理阶段删除 |
| 缩进 | 约定俗成，建议 4 空格或 2 空格，风格统一 |
| 标识符 | 字母数字下划线，不能以数字开头，区分大小写 |
| 关键字 | 保留词，不能用作标识符，C++20 已超过 90 个 |
| 分号 | 语句结束标志，if/for/while 后不要加分号 |
| 预处理 | `#` 开头，编译前执行，文本替换操作 |
| 程序结构 | 头文件 → using → 类/函数 → main |

## 课后练习

1. **注释练习**：在代码中添加多行注释和单行注释，说明 `#define` 和 `constexpr` 的区别
2. **宏练习**：编写程序，用 `#define` 定义宏计算两数之和，再用 `constexpr` 实现相同功能，比较两者差异
3. **条件编译练习**：用 `#if/#else/#endif` 实现 debug 和 release 两种编译模式
4. **参数练习**：将程序参数 `argc/argv` 的用法封装为一个函数，打印所有传入参数
5. **语法练习**：研究 C++17 `if (init; condition)` 语法，编写示例验证其作用域控制效果
