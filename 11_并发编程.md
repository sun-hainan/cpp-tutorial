## 1. 线程基础：std::thread

### 生活类比

> **线程**就像一条**流水线**——主线程是主装配线，可以创建多条支线（子线程）同时工作，加快整体进度。

### 概念讲解

```cpp
auto worker = [](int id) {
    std::cout << "线程 " << id << " 开始执行" << std::endl;
    std::this_thread::sleep_for(std::chrono::milliseconds(100));
    std::cout << "线程 " << id << " 执行完毕" << std::endl;
};

std::vector<std::thread> threads;
for (int i = 0; i < 3; ++i) {
    threads.emplace_back(worker, i);
}

for (auto& t : threads) {
    t.join();  // 主线程阻塞等待
}

// hardware_concurrency：CPU 核心数
std::cout << "硬件并发级别: " << std::thread::hardware_concurrency() << std::endl;
```

### 关键概念

| 方法 | 说明 |
|------|------|
| join() | 等待线程完成，主线程阻塞 |
| detach() | 线程在后台运行（守护线程） |
| hardware_concurrency() | 返回 CPU 核心数 |

## 2. mutex：线程同步

### 生活类比

> **mutex** 就像一个**公共厕所的锁**——进去的人（获取锁）把门锁上，其他人只能在外面等（阻塞）。

### 概念讲解

```cpp
std::mutex mtx;
int shared_counter = 0;

// std::lock_guard：RAII 加锁
{
    std::lock_guard<std::mutex> lock(mtx);  // 自动加锁
    ++shared_counter;
}  // 析构时自动解锁

// std::unique_lock：更灵活的 RAII 锁
{
    std::unique_lock<std::mutex> lock(mtx, std::defer_lock);
    lock.lock();
    ++shared_counter;
    lock.unlock();  // 可提前解锁
    lock.lock();
    ++shared_counter;
}

// std::timed_mutex：带超时的锁
std::timed_mutex tmtx;
if (tmtx.try_lock_for(std::chrono::milliseconds(100))) {
    std::cout << "获取锁成功" << std::endl;
    tmtx.unlock();
} else {
    std::cout << "获取锁超时" << std::endl;
}
```

## 3. 死锁：互相等待的"僵局"

### 生活类比

> **死锁**就像两个司机**互相让路**——A 让 B 先过，B 让 A 先过，结果谁也走不了。

### 死锁四要素

```
1. 互斥：资源只能被一个线程持有
2. 持有并等待：线程持有资源，还想要更多
3. 不可抢占：资源不能被强制释放
4. 循环等待：A 等 B，B 等 A
```

### 避免死锁的方法

```cpp
// 原则1：固定加锁顺序（所有线程按相同顺序获取锁）
std::mutex mtx_a, mtx_b;

auto process_a = [&]() {
    std::lock_guard<std::mutex> lock_a(mtx_a);  // A 先
    std::this_thread::sleep_for(std::chrono::milliseconds(10));
    std::lock_guard<std::mutex> lock_b(mtx_b);  // B 后
};

auto process_b = [&]() {
    std::lock_guard<std::mutex> lock_a(mtx_a);  // A 先（与 A 一致！）
    std::this_thread::sleep_for(std::chrono::milliseconds(10));
    std::lock_guard<std::mutex> lock_b(mtx_b);  // B 后
};

// 原则2：std::lock 原子获取多个锁
// std::lock(mtx_a, mtx_b);  // 不会死锁

// 原则3：C++17 scoped_lock
// std::scoped_lock lock(mtx_a, mtx_b);
```

## 4. 原子操作：无锁并发

### 生活类比

> **原子操作**就像**原子弹**——不可分割，CPU 保证整个操作一步完成，不会有中间状态被其他线程看到。

### 概念讲解

```cpp
std::atomic<int> atomic_counter(0);

// 基本操作
atomic_counter.store(42);       // 原子写入
int val = atomic_counter.load(); // 原子读取

// 原子读-改-写
int old = atomic_counter.fetch_add(1);  // 返回旧值

// CAS (Compare-And-Swap)
int expected = 43;
bool exchanged = atomic_counter.compare_exchange_strong(expected, 50);
```

### 内存顺序

| 顺序 | 说明 | 性能 |
|------|------|------|
| memory_order_seq_cst | 顺序一致（默认） | 最安全，最低 |
| memory_order_acquire | 获取屏障 | - |
| memory_order_release | 释放屏障 | - |
| memory_order_relaxed | 无同步要求 | 最快 |

## 5. 条件变量：线程间通信

### 生活类比

> **条件变量**就像**外卖送达通知**——你（等待线程）在等通知，有人（通知线程）按下通知按钮，你才去取餐。

### 概念讲解

```cpp
std::mutex cv_mutex;
std::condition_variable cv;
bool data_ready = false;

// 等待线程
auto waiter = [&]() {
    std::unique_lock<std::mutex> lock(cv_mutex);
    cv.wait(lock, [] { return data_ready; });  // 条件为真才返回
    std::cout << "数据已准备好" << std::endl;
};

// 通知线程
auto notifier = [&]() {
    {
        std::lock_guard<std::mutex> lock(cv_mutex);
        data_ready = true;
    }
    cv.notify_one();  // 通知一个等待线程
};
```

### 生产者-消费者示例

```cpp
std::queue<int> queue;
std::mutex queue_mutex;
std::condition_variable queue_cv;

auto producer = [&]() {
    for (int i = 0; i < 5; ++i) {
        {
            std::lock_guard<std::mutex> lock(queue_mutex);
            queue.push(i);
        }
        queue_cv.notify_one();
    }
};

auto consumer = [&]() {
    while (true) {
        std::unique_lock<std::mutex> lock(queue_mutex);
        queue_cv.wait(lock, [&]{ return !queue.empty(); });
        int val = queue.front(); queue.pop();
        std::cout << "消费: " << val << std::endl;
        if (val == 4) break;
    }
};
```

## 6. future/promise 与 async

### 生活类比

> **future/promise** 就像**快递柜**——寄件人（promise）把东西放进去，收件人（future）随时可以去取。

### 概念讲解

```cpp
// std::promise + std::future
std::promise<int> promise;
std::future<int> future = promise.get_future();

auto worker = [&](std::promise<int>& p) {
    std::this_thread::sleep_for(std::chrono::milliseconds(100));
    p.set_value(42);  // 设置结果
};

std::thread t(worker, std::ref(promise));
int result = future.get();  // 阻塞直到结果就绪
t.join();

// std::async：更高级的异步调用
auto async_task = [](int x) -> int {
    return x * 2;
};

std::future<int> f1 = std::async(std::launch::async, async_task, 21);
std::future<int> f2 = std::async(std::launch::deferred, async_task, 10);

std::cout << f1.get() << std::endl;  // 42（可能在另一线程执行）
std::cout << f2.get() << std::endl;  // 20（调用 get 时执行）
```

## 7. 线程池

### 概念讲解

```cpp
class ThreadPool {
public:
    ThreadPool(size_t n_threads) : stop_(false) {
        for (size_t i = 0; i < n_threads; ++i) {
            workers_.emplace_back([this] {
                while (true) {
                    std::function<void()> task;
                    {
                        std::unique_lock<std::mutex> lock(mutex_);
                        cv_.wait(lock, [this]{ return stop_ || !tasks_.empty(); });
                        if (stop_ && tasks_.empty()) return;
                        task = std::move(tasks_.front());
                        tasks_.pop();
                    }
                    task();
                }
            });
        }
    }

    template<typename F>
    void enqueue(F&& f) {
        {
            std::unique_lock<std::mutex> lock(mutex_);
            tasks_.emplace(std::forward<F>(f));
        }
        cv_.notify_one();
    }

    ~ThreadPool() {
        { std::unique_lock<std::mutex> lock(mutex_); stop_ = true; }
        cv_.notify_all();
        for (auto& w : workers_) w.join();
    }

private:
    std::vector<std::thread> workers_;
    std::queue<std::function<void()>> tasks_;
    std::mutex mutex_;
    std::condition_variable cv_;
    bool stop_;
};
```

## 本章小结

| 概念 | 核心要点 |
|------|---------|
| std::thread | join 等待，detach 后台运行 |
| mutex | lock_guard RAII 加锁，unique_lock 更灵活 |
| 死锁 | 固定顺序/std::lock/scoped_lock 避免 |
| atomic | 无锁操作，fetch_add/CAS |
| 条件变量 | wait/notify_one/notify_all |
| future/promise | 线程间传递结果，get 阻塞 |
| async | 最简洁的异步调用方式 |
| 线程池 | 复用线程，避免频繁创建/销毁开销 |
